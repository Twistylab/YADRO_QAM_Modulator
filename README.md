# QAM Modulator BER Analysis

Проект моделирует цифровую систему связи с модуляциями **QPSK**, **16-QAM** и **64-QAM** при передаче сигнала через канал с **AWGN** и сравнивает качество приёма по метрике **BER** для разных значений **SNR**.

Основная цель проекта - исследовать, как меняется **Bit Error Rate** при увеличении отношения сигнал/шум и усложнении схемы модуляции.

## Возможности

- генерация случайного битового потока;
- QPSK / QAM16 / QAM64-модуляция;
- нормировка средней мощности созвездия;
- Gray-кодирование компонент I/Q;
- добавление комплексного белого гауссовского шума (**AWGN**);
- демодуляция по ближайшей точке созвездия;
- расчёт **BER** для каждого значения **SNR**;
- сохранение результатов в CSV-файлы;
- построение графиков:
  - **BER vs SNR**;
  - **BER vs noise variance**.

## Назначение файлов

### `qam_modulator.cpp`

Главный вычислительный модуль проекта на C++.

Содержит:

- генерацию случайного битового вектора;
- реализацию AWGN-канала;
- QPSK / QAM16 / QAM64-модулятор;
- QPSK / QAM16 / QAM64-демодулятор;
- расчёт **BER**;
- сохранение результатов моделирования в CSV-файлы.

Ключевые классы и функции:

- `AWGNAdder` - добавляет комплексный гауссовский шум к переданному сигналу;
- `ModulatorGrey` - преобразует битовый поток в комплексные символы выбранной модуляции;
- `DemodulatorGrey` - восстанавливает биты по ближайшей точке сигнального созвездия;
- `generate_bits_vector` - генерирует случайный битовый поток;
- `compute_ber` - считает долю ошибочно принятых битов.

По умолчанию программа перебирает значения **SNR** от `0` до `20 dB` с шагом `1 dB`.
Для каждого значения SNR генерируется `bits_per_symbol * 1_000_000` бит.

### `modulation_analyzer.py`

Основной Python-сценарий для запуска эксперимента.

Скрипт:

- запрашивает у пользователя список модуляций;
- проверяет, что введены только поддерживаемые типы: `QPSK`, `QAM16`, `QAM64`;
- запускает C++-симулятор `qam_modulator.exe`;
- строит графики по полученным CSV-файлам;
- сохраняет итоговое изображение в `ber_vs_snr_and_variance.png`.

Пример ввода:

```text
QPSK,QAM16,QAM64
```

### `plot_ber.py`

Модуль визуализации результатов.

Содержит функцию `plot_ber`, которая:

- читает файлы `ber_vs_snr_<MODULATION>.csv`;
- строит график **BER vs SNR**;
- переводит SNR в дисперсию шума по формуле `10^(-SNR / 10)`;
- строит график **BER vs noise variance**;
- использует логарифмическую шкалу по оси BER;
- отбрасывает значения ниже `1e-6`, чтобы график оставался читаемым.

## Как работает модель

Общий пайплайн симуляции:

1. Выбор типа модуляции: `QPSK`, `QAM16` или `QAM64`.
2. Определение количества бит на символ.
3. Генерация случайного битового потока.
4. Разделение битов на I/Q-компоненты.
5. Преобразование битов в точки QAM-созвездия.
6. Нормировка средней мощности созвездия.
7. Добавление комплексного AWGN-шума.
8. Демодуляция принятого сигнала по минимальному евклидову расстоянию.
9. Сравнение переданных и принятых битов.
10. Расчёт **BER**.
11. Повторение эксперимента для всех значений **SNR**.
12. Сохранение результатов в CSV.
13. Построение итоговых графиков.

## Математическое описание модуляции

В проекте используется квадратная QAM-модуляция размера \(M\), где:

\[
k = \log_2 M
\]

— количество бит на один комплексный символ, а

\[
L = \sqrt{M}, \qquad m = \frac{k}{2}
\]

— количество уровней на одной оси созвездия и количество бит на одну компоненту сигнала.

Каждый блок из \(k\) бит разбивается на две независимые группы: для синфазной компоненты \(I\) и квадратурной компоненты \(Q\):

\[
\mathbf{b}^{(I)} = (b_0, b_2, \dots, b_{k-2}),
\]

\[
\mathbf{b}^{(Q)} = (b_1, b_3, \dots, b_{k-1}).
\]

После этого для каждой компоненты применяется одна и та же функция отображения \(\alpha(\cdot)\). Комплексный передаваемый символ задаётся как:

\[
s = I + jQ = \alpha(\mathbf{b}^{(I)}) + j\alpha(\mathbf{b}^{(Q)}).
\]

### Нормировка созвездия

До нормировки уровни каждой оси задаются нечётными значениями:

\[
\mathcal{A}_L = \{L - 1, L - 3, \dots, -(L - 3), -(L - 1)\}.
\]

Средняя энергия комплексного символа вычисляется как:

\[
E_s = \frac{1}{M}\sum_{I \in \mathcal{A}_L}\sum_{Q \in \mathcal{A}_L}(I^2 + Q^2).
\]

В коде каждая координата делится на \(\sqrt{E_s}\), поэтому средняя мощность нормированного созвездия становится равной единице:

\[
\frac{1}{M}\sum_{s \in \mathcal{S}} |s|^2 = 1.
\]

Для поддерживаемых модуляций получаются следующие значения нормировочного множителя:

| Модуляция | \(M\) | \(E_s\) | Нормировка |
|---|---:|---:|---:|
| QPSK | 4 | 2 | \(\sqrt{2}\) |
| QAM16 | 16 | 10 | \(\sqrt{10}\) |
| QAM64 | 64 | 42 | \(\sqrt{42}\) |

### Формулы отображения битов в координаты

Для **QPSK** на каждую ось приходится один бит. Координата вычисляется по формуле:

\[
\alpha_4(b_0) = \frac{1 - 2b_0}{\sqrt{2}}.
\]

Отсюда:

\[
b_0 = 0 \Rightarrow \alpha_4 = \frac{1}{\sqrt{2}},
\qquad
b_0 = 1 \Rightarrow \alpha_4 = -\frac{1}{\sqrt{2}}.
\]

Для **16-QAM** на каждую ось приходится два бита. Используется Gray-отображение:

\[
\alpha_{16}(b_0, b_1) =
\frac{1 - 2b_0 + 2b_1 - 4b_0b_1}{\sqrt{10}}.
\]

Таблица соответствия для одной координаты:

| \(b_0\) | \(b_1\) | Ненормированный уровень | Нормированный уровень |
|---:|---:|---:|---:|
| 0 | 0 | \(+1\) | \(+1 / \sqrt{10}\) |
| 1 | 0 | \(-1\) | \(-1 / \sqrt{10}\) |
| 0 | 1 | \(+3\) | \(+3 / \sqrt{10}\) |
| 1 | 1 | \(-3\) | \(-3 / \sqrt{10}\) |

Для **64-QAM** на каждую ось приходится три бита. Координата вычисляется как:

\[
\alpha_{64}(b_0, b_1, b_2) =
\frac{
3 - 6b_0 + 2b_1 - 2b_2 - 4b_0b_1 + 4b_0b_2 + 4b_1b_2 - 8b_0b_1b_2
}{\sqrt{42}}.
\]

Таблица соответствия для одной координаты:

| \(b_0\) | \(b_1\) | \(b_2\) | Ненормированный уровень | Нормированный уровень |
|---:|---:|---:|---:|---:|
| 0 | 0 | 0 | \(+3\) | \(+3 / \sqrt{42}\) |
| 1 | 0 | 0 | \(-3\) | \(-3 / \sqrt{42}\) |
| 0 | 1 | 0 | \(+5\) | \(+5 / \sqrt{42}\) |
| 1 | 1 | 0 | \(-5\) | \(-5 / \sqrt{42}\) |
| 0 | 0 | 1 | \(+1\) | \(+1 / \sqrt{42}\) |
| 1 | 0 | 1 | \(-1\) | \(-1 / \sqrt{42}\) |
| 0 | 1 | 1 | \(+7\) | \(+7 / \sqrt{42}\) |
| 1 | 1 | 1 | \(-7\) | \(-7 / \sqrt{42}\) |

Одна и та же функция \(\alpha\) применяется отдельно к битам \(I\)- и \(Q\)-компоненты. Поэтому, например, для 16-QAM комплексный символ имеет вид:

\[
s = \alpha_{16}(b_0, b_2) + j\alpha_{16}(b_1, b_3).
\]

А для 64-QAM:

\[
s = \alpha_{64}(b_0, b_2, b_4) + j\alpha_{64}(b_1, b_3, b_5).
\]

### Модель канала и демодуляция

После модуляции к символу добавляется комплексный AWGN-шум:

\[
r = s + n,
\]

где

\[
n = n_I + jn_Q,
\qquad
n_I, n_Q \sim \mathcal{N}\left(0, \frac{\sigma^2}{2}\right).
\]

Дисперсия шума связана с SNR так:

\[
\sigma^2 = 10^{-\frac{SNR_{dB}}{10}}.
\]

Демодулятор выбирает ближайшую точку созвездия по евклидову расстоянию:

\[
\hat{s} = \arg\min_{s_i \in \mathcal{S}} |r - s_i|.
\]

После восстановления битов считается эмпирическая вероятность битовой ошибки:

\[
BER = \frac{1}{N}\sum_{i=0}^{N-1} \mathbb{1}\{b_i \ne \hat{b}_i\}.
\]

## Модуляции

Проект поддерживает три типа модуляции:

| Тип | Размер созвездия | Бит на символ |
|---|---:|---:|
| `QPSK` | 4 | 2 |
| `QAM16` | 16 | 4 |
| `QAM64` | 64 | 6 |

Для каждой модуляции используется квадратное QAM-созвездие с раздельным кодированием I- и Q-компонент.

## Канал

В проекте используется канал с аддитивным белым гауссовским шумом:

```text
rx_signal = tx_signal + noise
```

Шум добавляется независимо в действительную и мнимую части комплексного сигнала.
Дисперсия шума рассчитывается по SNR:

```text
noise_variance = 10^(-SNR / 10)
```

## Особенности реализации

- Модулятор и демодулятор реализованы без сторонних C++-библиотек.
- Созвездия нормируются по средней мощности.
- Демодуляция выполняется через полный перебор точек созвездия и выбор ближайшей по евклидову расстоянию.
- Для каждой модуляции и каждого значения SNR генерируется новый случайный битовый поток.
- Python используется только для оркестрации запуска и визуализации результатов.

## Ограничения

- Поддерживаются только `QPSK`, `QAM16` и `QAM64`.
- Моделируется только AWGN-канал без замираний, кодирования, интерливинга и эквализации.
- Готовый `qam_modulator.exe` из репозитория может не запускаться вне Windows.
- `plot_ber.py` рассчитан максимум на три модуляции, потому что список цветов содержит три значения.
